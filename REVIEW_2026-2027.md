### Co-requisite / prerequisite mis-parse — bulk fix (April 2026)

Chris's stress-test surfaced a systematic data bug: during the bake
from the catalogue PDF, several "Recommended" entries were transcribed
as **Co-requisites** or **Pre-requisites** in the CSV. Examples he
flagged: PHY3001 (Quantum Mechanics) listed MAT2006 as a co-req when
the catalogue has MAT2006 under *Recommended*; PRA2019 (Natural
Science Illustration) listed PRA2015 as a co-req when it's actually
just a *Recommendation*.

Audited all four period CSVs against the catalogue markdown and
flipped **18 rows** where the CSV asserted a requirement but the
catalogue's first bullet in that section says "None":

**Co-requisites flipped to FALSE (10):** PHY3001, PHY3009, PHY2012,
PRA2019, PRA3503 (P1) · PHY3004 (P2) · BIO2005, CHE3011 (P4) ·
CHE3012, PHY2005 (P5).

PHY2006 / PRA2006 (P1) were initially flipped to FALSE in the same
pass (catalogue markdown said "None"), but the April 2026 **final**
frame-schedule CSVs show ◊ for both, so we reinstated the pair
(PHY2006 ↔ PRA2006) — the frame-schedule is the authoritative
source here.

**Prerequisites flipped to FALSE (6):** INT3002 (P1) · BIO3010 (P4) ·
PHY2007, INT2009, PRA2009, PRA2023 (P5).

The *Recommended* info is still visible in the full catalogue view —
it just no longer surfaces as a fake warning in the planner summary.

**Remaining ambiguous cases worth a human eyeball:**

- CHE2010 ↔ PRA2034 (P5): cross-referenced as co-reqs in the CSV.
  Catalogue uses `•` bullets (no ✓ prefix) so my auto-audit flagged
  them; confirmed via the P5 Frame Schedule PDF that the ◊ markers
  line up — the pair is legitimate. Left untouched.

**Multi-code co-requisites (now supported):**

PRA1101 and PRA1102 have catalogue-declared co-reqs with two codes each:

- PRA1101 (P1) → `CHE1101 + PHY1101`
- PRA1102 (P2) → `BIO1101 + INT1101`

The `CorequisiteOf` field previously only held a single code, so these
co-reqs were under-specified (a student picking PRA1102 + BIO1101 +
something-else would pass the check even though INT1101 is missing).

The schema now accepts the same `+` (AND) / `/` (OR) grammar as
`PrerequisiteCodes`. The CSV values above have been updated, and a
new `checkCoreqMet(expr, presentCodes)` helper in the planner
evaluates them against the set of currently-selected codes +
transcript-passed codes (auto-waive). All existing single-code co-reqs
continue to work unchanged (a single code is just a degenerate case
of the expression grammar).

A one-off audit script at bake time that cross-checks every
`Corequisite=TRUE` / `Prerequisite=TRUE` against the catalogue would
prevent this class of error from re-entering.

### Alignment with final April 2026 frame schedules

The initial CSVs were built from the "almost final" draft. When the
official frame schedules arrived (uploaded as `files (20)/`) they
used a different layout (Type/Row/TimeSlot columns, `◊`/`▲` pictogram
flags, new `ModuleGroup` column, no AM/PM granularity, no specific
co-req/pre-req codes). Rather than swap formats wholesale (and lose
the specific pairings we'd derived from the catalogue), we kept our
richer format and patched in the substantive differences:

- **Island Studies** titles shortened to the official form
  ("Life and Landscape", "The Mediterranean Region", "Fieldwork and
  Data Collection"); the bundle label ("Mediterranean Island Studies")
  in the planner supplies the overall context.
- **George Mescher** replaces the previous "Mesher" spelling for the
  Island Studies coordinators (matches the catalogue + frame schedule).
- INT3202 and PRA3201 gained the entry pre-requisite
  (`PRA2009/INT1007/INT2012`, shared with INT3201) — the new schedule
  flags all three as `▲`.
- **PHY2006 ↔ PRA2006** co-req pair reinstated (see above).
- **NEU2005** pre-requisite removed — the new frame schedule has no
  `▲` flag on it, so the previous `NEU1001/BIO2007` prereq was either
  dropped or never official.
- **PRA2031** (Python Programming Language, P4) pre-requisite removed
  for the same reason.
- **INT3007** coordinator trimmed to "Martina Summer-Kutmon" (Judith
  Peters removed, matching the official schedule).
- Typos fixed: "Alexandr" → "Alexander" (PRA2030, PRA3023);
  "Statistical Modeling" → "Statistical Modelling".
- Location markers (`*`, `**`) stripped from titles — they were the
  "Campus Venlo" / "Chemelot" / "with UCM" notes and are now only in
  the catalogue descriptions. If the front-page summary should show
  them, adding a `Notes` column + a small chip in the card would be
  the clean fix.

One regression to be aware of: **the official CSV format does not
list the specific target codes** for co-reqs or pre-reqs — only a
`◊`/`▲` flag that something exists. Our format retains the explicit
codes (derived from the catalogue markdown), which is what lets the
planner actually check satisfaction. If the team ever adopts the new
CSV format as canonical, we'd need to either (a) keep the catalogue
markdown as the secondary source for specific codes, or (b) extend
the new format with target columns.

For practicals **already in the 2025-2026 P5 CSV**, I reused their known-good
`Days` values. For practicals **only in P1 / P2 / P4** (or new this year),

default of both AM and PM per listed day. This may cause over-reporting of
conflicts in rare cases. Give me AM/PM specifics when you have them and
